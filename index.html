Project Goal

Objective

This project implements a real-time electrocardiogram (EKG) signal acquisition and processing system on an FPGA. Using a Terasic DE1-SoC (Cyclone V) board and a SparkFun AD8232 single-lead heart rate monitor, the system captures analog cardiac signals through the onboard LTC2308 12-bit SAR ADC, processes them through a digital signal processing (DSP) pipeline implemented entirely in VHDL, and streams the processed data to a PC-based Python visualizer for real-time waveform display and heart rate calculation.

Background

Introduction

Electrocardiogram (EKG/ECG) signals are among the most critical biomedical measurements, providing insight into cardiac rhythm, conduction abnormalities, and overall heart health. Commercial EKG systems rely on specialized mixed-signal ASICs for signal conditioning and DSP. This project explores an alternative approach: using an FPGA to perform the entire digital processing pipeline in hardware, offering deterministic timing, parallelism, and the flexibility to modify the processing chain without hardware changes.

The system is organized into three layers:

  1. Analog front-end: The AD8232 single-lead heart rate monitor amplifies and filters the raw electrode signal, producing an analog output suitable for ADC digitization.
  2. FPGA digital back-end: The Cyclone V FPGA drives the LTC2308 ADC via SPI at 360 samples/second, then routes the 12-bit samples through a configurable DSP pipeline (DC blocking, 50/60 Hz notch filter, bandpass FIR, QRS peak detection).
  3. PC visualization: A Python application reads processed samples over the USB-Blaster JTAG interface and renders a real-time scrolling waveform with calculated BPM.

Background Math & Theory

EKG Signal Characteristics

  • Frequency range: 0.5 Hz – 40 Hz (diagnostic bandwidth)
  • Amplitude: ~1 mV at the skin, amplified to ~1–3 V by the AD8232
  • Key features: P-wave, QRS complex, T-wave
  • Sampling rate: 360 Hz (matching the AHA EKG database standard)
💓

Labeled diagram of a single EKG heartbeat: P-wave, QRS complex, T-wave, PR interval, QT interval, ST segment.

ADC Sampling

The LTC2308 is a 12-bit SAR ADC with SPI. Key parameters: 4096 levels, 0–4.096 V range, 500 ksps max (we use 360 sps). At 360 Hz, the Nyquist frequency is 180 Hz — well above the 40 Hz EKG bandwidth.

Logical Structure

The FPGA uses a synchronous pipeline in a single 50 MHz clock domain:

  1. sample_clock divides 50 MHz → 360 Hz tick.
  2. Each tick triggers adc_spi_driver for an SPI transaction.
  3. 12-bit result flows through dc_block → notch_5060 → bandpass_fir.
  4. qrs_detect identifies R-peaks for BPM.
  5. jtag_streamer sends samples to PC over JTAG UART.
📌All modules share a common clk / reset_n interface with valid strobes for handshaking.

Hardware/Software Tradeoffs

DSP in VHDL (not on the HPS ARM core) was chosen for:

  • Deterministic latency: Fixed sample-to-output delay, no OS jitter.
  • Parallelism: Multiple filter stages run concurrently.
  • Learning: The goal was FPGA-based DSP exploration.

Python handles visualization because PyQtGraph provides real-time plotting and the JTAG UART requires zero additional hardware.

Add additional tradeoffs: fixed-point vs floating-point, filter order vs resource usage, JTAG bandwidth.

Relationship to Standards

  • AHA Database Standard: 360 Hz sampling matches the AHA EKG database for future test-vector validation.
  • IEC 60601-2-47: Pipeline designed with awareness of the 0.05–40 Hz monitoring bandwidth.
  • SPI: ADC communication follows SPI Mode 0 (CPOL=0, CPHA=0).

Part 3

FPGA Design

ADC SPI Driver

The adc_spi_driver implements a bit-banged SPI master for the LTC2308. Triggered by the sample clock, it:

  1. Asserts ADC_CONVST high for one clock cycle.
  2. Clocks out a 6-bit channel config on ADC_DIN (CH0).
  3. Simultaneously clocks in 12 bits on ADC_DOUT.
  4. Asserts sample_valid when the result is ready.

Safety Considerations

⚠️ This project is NOT a medical device. It is an educational prototype and must not be used for clinical diagnosis.
  • AD8232 provides built-in input protection and right-leg drive.
  • Electrode connections are low-voltage (3.3V), low-current — no shock hazard.
  • Electrically isolated from mains when running on USB power.
